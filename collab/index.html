Architecture

Collab is built on a multi-tenant architecture where each organization operates within its own isolated server. Data remains partitioned at the account level while access is controlled through role-based licenses.

  • System Server — Shared governance layer containing core platform logic.
  • Public Server — Shared collaboration layer for approved public workflows and data.
  • Client Server — Dedicated operational environment owned and managed by an organization.

Within each client server, organizations can create multiple accounts to separate business units, projects, legal entities, or reporting periods. Accounts may be perpetual for long-lived records or periodic for period-controlled transactional activity.

For example, if Transaction T references Contact X, a user with access to both records can view both Transaction T and Contact X. A user with access only to Transaction T can view the transaction but cannot access Contact X. References do not override security permissions.

Licensing

Collab uses a subscription-based licensing model that governs access across servers and accounts. Each license defines what users can view, create, update, and administer.

  • Viewer — Read-only access.
  • Editor — Create and update records.
  • Owner — Full administrative control.
